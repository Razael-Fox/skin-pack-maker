# Artifact vs. Workspace Source Code Files in Antigravity

When working with files in Google Antigravity, it is critical to distinguish between **Workspace Source Code Files** and **Agent Artifacts**. Misidentifying the target type when invoking file writing tools will trigger path validation failures.

## 1. File Categorization

### Workspace Source Code Files

These are the files that make up the actual software application, library, or website being built/maintained by the user.

- **Location:** Typically inside the active workspace directory (e.g., `/data/data/com.termux/files/home/skin-pack-maker/src/...`).
- **Purpose:** Source code, configuration, styles, assets.
- **Tool Rule:** Never include the `ArtifactMetadata` parameter when calling `write_to_file`, `replace_file_content`, or `multi_replace_file_content` for these files.

### Agent Artifacts

These are markdown documents, reports, or logs generated by the agent for the user's review.

- **Location:** Always inside the designated conversation artifacts directory (e.g., `/data/data/com.termux/files/home/.gemini/antigravity-cli/brain/<conversation-id>/`).
- **Purpose:** Project plans, architectural analysis, checklists, research notes.
- **Tool Rule:** You MUST include the `ArtifactMetadata` parameter when creating or updating these files.

## 2. Key Error Symptoms

If you try to write/modify a workspace source code file (e.g., `/data/data/com.termux/files/home/skin-pack-maker/src/lib/mcpackParser.ts`) and mistakenly supply `ArtifactMetadata`, you will encounter the following error:

```
Error Message: model output error: invalid tool call error (invalid_args)
/path/to/workspace/file is not a valid artifact path; artifacts must be in /path/to/agent/brain/<conversation-id>/
```

## 3. Resolution Steps

To resolve the error:

1. Re-run the tool invocation without the `ArtifactMetadata` block for project source code files.
2. Confirm the destination path is inside the appropriate directory (workspace vs. brain logs).
